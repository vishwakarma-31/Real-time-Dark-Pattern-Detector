Week 2: Core Multi-Modal Detection Engine + Fusion + Evaluation
Day 8 Date: __ / __ / 20__
Tasks Planned Implement the NLP detector using GPT-4o with a structured few-shot system prompt. Define the 6-category ENUM and JSON response format.
Tasks Completed Wrote runNlpDetector() in backend/services/analyzeService.js. Designed the GPT-4o system prompt specifying JSON-only output with the schema: { detected: boolean, patterns: [{ category: ENUM, confidence: 0.0-1.0, evidenceText: string }] }. Used response_format: { type: 'json_object' } to eliminate markdown wrapping. Constructed the textCorpus string by joining DOM snapshot arrays (buttons, modals, checkboxes) with pipe separators. Tested against Amazon checkout text samples and correctly identified "Subscribe & Save" as forced_continuity (confidence 0.91) and "Only 3 left in stock" as fake_countdown (confidence 0.87).
Problems Faced GPT-4o occasionally returned patterns with confidence values above 1.0 due to probabilistic output, which broke downstream fusion math.
Solutions Applied Added post-processing: patterns.map(p => ({ ...p, confidence: Math.min(1.0, Math.max(0.0, p.confidence)) })) before returning from the detector. Added input validation to reject any pattern object missing a required field.
References Consulted OpenAI API documentation (response_format, JSON mode). Few-shot prompting guide from OpenAI Cookbook.
